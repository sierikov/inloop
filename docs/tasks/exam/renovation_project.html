Renovation

A renovation project supports planning material costs for a RenovationObject which comprises various partial StructuredObjects such as rooms or apartments. The aggregation between StructuredObject and RenovationObject must be implemented with a java.util.Set.

The method addMaterialReq() is given a java.util.Map<String, Integer> which maps Material names to integers. The number associated with a certain Material‘s name specifies how many units of that Material are needed for the RenovationObject.
StructuredObject.addMaterialReq() gathers every Material requirement for the RenovationObject, adds them to a copy of the given Map and returns this updated Map. It is required that every Surface has a Material assigned to it in order to execute Surface.addMaterialReq() properly.

The method getMaterialReq() calculates the amount of material required for a given Surface s. Possible materials are floor coverings (Flooring) and coatings (Paint).

Besides having a name and price per (selling) unit, materials have certain additional attributes:

  1. Flooring
    1. widthOfFlooring (in meters)
    2. Since you can buy only a whole number of floor coverings, getMaterialReq() may have to round up. For this calculation, there is a certain (exclusive) tolerance value specified by the static attribute limit (use 0.02 as value). To calculate how many panels you need for a given Surface, divide the Surface‘s area by widthOfFlooring and round up if needed (mind the tolerance value!).
  2. Paint
    1. noOfCoats (the number of layers of paint to be applied to the Surface)
    2. noOfSqMPerLiter (the number of square meters that can be painted with one liter of the paint)
    3. Paint can be bought in buckets of half a liter. For the calculation of the number of paint buckets required, there is a certain (exclusive) tolerance value (in liters) specified by the static attribute limit (use 0.02 as value). To calculate the number of paint buckets required for a given Surface, multiply the Surface‘s area with noOfCoats, divide the result by noOfSqMPerLiter and round up if needed (mind the tolerance value!). Multiply the result by 2 (to get the exact amount of half-liter buckets).

Task

Before you start to write code, try to find design patterns that are helpful for the implementation of the classes. Afterwards, implement the renovation project classes!

Hints

  1. Check method arguments and handle invalid values. For instance, an argument of value null should result in a NullPointerException. Likewise, an empty String or a negative price should result in a IllegalArgumentException.

  2. Since double types have finite precision it will be necessary for you to round values, or otherwise the limit calculation won’t work. In Java, the following holds true: 1.52 % 0.5 != 0.02 (instead it is 0.020000000000000018). 

    inaccurateCalculation = 1.52 % 0.5;
fixedCalculation = Math.round(inaccurateCalculation * 100) / 100;
