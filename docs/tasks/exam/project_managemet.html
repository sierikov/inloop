Project Management

Your task is to implement a new version of a project management environment. A project consists of several tasks and deliverables (documents or products, which have to be delivered). Furthermore, the old version’s functionality of getting information about a project’s total costs and required time must be included.

You need to implement the following classes. They are also shown in the class diagram below.

  1. The class Project represents a single project. Project provides several operations to retrieve the necessary information about a project. The actual structure of a project is represented by the classes ProjectItem, Task and Deliverable. A project cannot be created without a valid rate, which is used for a project’s main task. The method allDeliverables() returns a Map, which maps (due) dates to the corresponding Deliverable objects.

  2. The class ProjectItem is the main part of a project’s structure. All project items have a name and a detailed description. Furthermore, there is a rate which describes the costs per time unit. The estimate costs are calculated by multiplying the required time with the rate and adding that value (rounded to a long) to the material costs. There are two different types of project items: tasks and deliverables.

  3. The class Task describes a task within a project. Each task may contain sub-tasks and/or deliverables. The material costs and required time of a task is the sum of the costs/time of all of the task’s sub-items.

  4. The class Deliverable describes a document or product which has to be delivered for the work on the associated project. A Deliverable object is characterized by a date of delivery, material cost and production time.

Task

Implement the project management explained above!

Implement a class Adapter, which maps the old interface IProject to the new Project class. Thus, you have to take a closer look at the old getDeliverables() method, which only returns all Deliverable objects of the project in a List.

Client is an unknown application which manages projects using only the interface IProject.

Hints
  1. Use the type long for the calculation of prices in cents, since double cannot accurately represent the base 10 multiples we use for money.
  2. The most important erroneous parameter values must be handled properly. Therefore, the parameter value null should result in a NullPointerException while others like empty Strings, zero or negative numbers (for prices) that do not make sense should cause an IllegalArgumentException.
