Predicate Iterator

Inloop / Tasks / Predicate Iterator

Predicate

A PredicateIterator<T> iterates over the elements of an arbitrary data iterable structure and provides only those elements which satisfy a certain predicate (while skipping all of the other elements).

Given is the description of a design model which uses the iterator design pattern (see the class diagram below):

The constructor of PredicateIterator<T> expects the following arguments:

  1. iter, a java.util.Iterator<T> for the actual data
  2. a description of the predicate consisting of
    1. pred, a Predicate<T> object that determines what kind of predicate to use
    2. argument, an object of type T that serves as value for the predicate

The method predicate() of the generic interface Predicate<T> checks whether an object element satisfies the predicate with the value argument (both are of type T). It returns true or false accordingly.

The design model contains three classes which are implementations of the interface Predicate<T> for the application to String objects. Note that element and argument are of type T extends String.

  1. PredicateStartsWith checks if element starts with the value argument.
  2. PredicateEndsWith checks if element ends with the value argument.
  3. PredicateLength checks if element has the length argument (since argument is of type
    T extends String, use Integer.parseInt() to get an int for the length check).
