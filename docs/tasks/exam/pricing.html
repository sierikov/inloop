Pricing

The UML class diagram shown below models a complex pricing logic that allows for various discounts on products. Two basic categories of discounts are available:

  1. Percentage discount on the catalog price (class PercentageDiscountPricing). The discount percentage to grant is passed to the constructor (0 <= percentage <= 100).
  2. Absolute discount on the catalog price (class AbsoluteDiscountPricing). Absolute discount is available only for products with a catalog price above a certain minimum price (threshold). The final product price may not fall below this threshold after granting the discount. Both discount and threshold are passed as arguments to the constructor.

The class Sale represents a certain product an its discounts. Therefore, the catalog price of the product (preDiscountTotal) is stored. The method getTotal() determines the final price of the product after all registered discounts have been applied. The actual discount calculation is performed in the method getTotal() of the classes implementing the interface ISalePricing.

Complex Pricing Structures

In general, there are cases in which several discounts can be applied at the same time, e.g. discounts for seniors or other special customers, loyalty bonus, deal of the day. While several discounts may be registered for a Sale object, only one can actually be applied because our pricing logic is very simple. A complex pricing logic (abstract class ComplexPricing) is responsible for selecting the discount to be applied. There are two concrete pricing logic implementations:

  1. The class BestForCustomerPricing calculates the selling price based on the largest discount for the customer.
  2. The class BestForStorePricing selects the discount that is best for the vendor (i.e. lowest total discount).

Before the final price of a product is determined, all discounts to chose from must be known to the selected pricing logic. Different discounts can be created by the static method Sale.createPricing(). The arguments of this method are:

  1. discountType: A value of the enumeration DiscountType that determines whether to use an absolute or percentage discount.
  2. percentage: In case of a percentage discount, this argument specifies the granted percentage. The two remaining arguments should be 0 in this case.
  3. discount and threshold: In case of an absolute discount these arguments describe the granted discount, as explained above. The percentage argument should be 0 in this case.

Finally, the desired pricing logic pricing (BestForCustomerPricing or BestForStorePricing) is being selected using setPricing().

To get an idea of how this whole pricing system is used, you can find an exemplary main class in the Resources section.

Hints

  1. he most important erroneous parameter values must be handled properly. Therefore, the parameter value null should result in a NullPointerException while others like empty Strings, zero or negative numbers (for prices) that do not make sense should cause an IllegalArgumentException.

  2. n the related exam task, all parameters and variables representing money are of type double to keep the task simple. Since double values are too inaccurate for monetary calculations and thus are not used in real world applications, you should use long instead of double (representing the corresponding amount of cents).
