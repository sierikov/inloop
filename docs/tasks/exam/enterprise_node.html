Enterprise Node

In this task, we are modeling the structure of enterprises with trees that consist of EnterpriseNodes. Possible node types within the tree are AbstractEnterpriseUnits as well as actual people within the enterprise (StaffMember).

The organizational units of enterprises are structured hierarchically: Holdings consist of companies, companies consist of divisions and divisions are separated into teams. The former three are AbstractUnits and can be assigned direct child nodes in the tree structure via add(). Adding a wrong unit (e.g. a Team to a Holding) should result in an IllegalArgumentException. The method remove() accordingly removes a given direct child node. Both methods return true if and only if they were supposed to actually perform their operation. For example, when trying to add a node that is already a direct child node, add() should return false and not add the node again.

At another level of the enterprise structure, there are teams as the fundamental enterprise units. Each team is led by a single StaffMember, the teamLeader. Additionally, the class StaffMember keeps track of the directSubordinates of the respective person. The method addDirectSubordinate() and removeDirectSubordinate() work analogous to AbstractUnit‘s add() and remove().

StaffMember objects can be compared to other objects of their kind by calling compareTo(). This method implements the behavior defined by the interface java.lang.Comparable<T> regarding the alphabetic order of the compared staff members’ names. Implementing this interface makes the class StaffMember more flexible allowing different data structures to store and manage its objects.

To iterate over the whole enterprise structure, we define an interface EnterpriseNodeIterator which extends the interface java.util.Iterator<T> by restricting the possible values of the type parameter T to implementations of EnterpriseNode. Apart from that, an EnterpriseNodeIterator‘s methods hasNext() and next() behave as defined by Iterator<T> (see its documentation in the Java API Documentation).

We want to focus on the StaffMemberIterator as one concrete iterator for the enterprise structure. Its constructor is given a Set of staff members (directSubordinates) and additionally determines all staff members below these directSubordinates recursively (findSubordinatesRecursively()). Eventually, the resulting set of staff members (allMembers) is sorted alphabetically by the name of the staff members (see Hints). Afterwards, the iterator methods mentioned above must be able to iterate over all of the directly and indirectly given staff members in order.

Hints
  1. The most important erroneous parameter values must be handled properly. Therefore, the parameter value null should result in a NullPointerException (where it makes sense) while others like empty Strings should cause an IllegalArgumentException.
  2. To check if an object is an instance of a certain class you can use the keyword instanceof.
  3. A TreeSet is a collection that, unlike a HashSet or a set in mathematical set theory, may be sorted. It still disallows duplicate elements.
