MyMatrix

Take a look at the following interface of a data structure representing a matrix.

import java.util.*;

public interface Matrix<T> {
    int getRowCount();
    int getColumnCount();
    int getObjectCount();
    int getDistinctObjectCount();
    Iterator<T> iterator();
    T get(int row, int column);
    T put(int row, int column, T value);
    boolean contains(T value);
}
Example

To understand how the data structure is supposed to work, take a look at the following example.

Matrix<String> m = new MyMatrix<String>();
String a = "a";
String b = "b";

m.put(0, 1, a);
m.put(1, 3, b);
m.put(2, 0, a);

The state of the matrix after executing that code looks like this:

    0       1       2       3
0   null    a       null    null
1   null    null    null    b
2   a       null    null    null

The following table shows the result of exemplary calls of methods of Matrix which should give you an impression of what these methods do.

getRowCount() 3

get(2, 0) a

Hints
  1. An entry of the matrix is defined as empty if it holds the value null or its row or column do not exist.
  2. The row and column counts are non-negative integers.
    1. The indices of the first row and the first column are 0.
    2. The methods getRowCount() and getColumnCount() do not count empty rows at the bottom edge or empty columns at the right edge of the matrix.
  3. The method getObjectCount() only counts the non-empty entries of the matrix.
  4. The method getDistinctObjectCount() only counts the number of different objects in the matrix.
  5. Implement the iterator (provided by MyMatrix.iterator()) with the iterator pattern.
    1. Implement the iterator (DepthFirstIterator) as inner class of MyMatrix.
    2. The iterator traverses the matrix first in depth, then in width, hence the name.
    3. Implement the methods next() and hasNext() according to the specification of the Iterator interface. The method remove() is not supported by the iterator.
    4. null objects are skipped by the iterator, so they are not provided by next().
  6. The method get(int row, int column) returns the object at the given row and column.
  7. The method put(int row, int column, T value) assigns the object value to the entry at position (row, column). The method returns the original object which was replaced by value. If nothing (null) was at the position previously, the method returns null.
  8. The method contains(T value) checks if the matrix contains the given object value of type T.
  9. Matrix should be generic and the type variable T determines the type of objects that can be put into the matrix. Save the entries in a java.util.Map.
    1. Use the class java.util.HashMap<K,V> for the matrix and create a class MatrixIndex for the positions.
    2. A MatrixIndex is defined as a pair of an int row and an int column. Create for both of them getters, but not setters. We want to leave our MatrixIndexs immutable.
    3. Don’t forget to override the methods MatrixIndex.equals() and MatrixIndex.hashCode().
