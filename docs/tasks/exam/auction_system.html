Auction System

In this task you should implement the functionality of an auction system, which should support two different types of auctions: An English auction and an all-pay auction. The auction system consists of six different classes:

  1. Auction: Each auction has a java.util.List of its auctioned items (allItems) and a list of all bidders (bidders).
  2. EnglishAuction and AllPayAuction: These are subclasses of an Auction.
  3. Item: An item has a name, a description and a minPrice. Each item has a java.util.List containing all bids on that item (allBids).
  4. Bid: Each bid has a price and a related bidder.
  5. Person: Only the name is stored for a person.
Sequence of an English auction

First an auctioneer opens (creates) an auction. Then he/she creates the auctioning items and adds each one to the allItems list (registerItem()). Once all items are registered, the bidders may place bids on them (addBid()). The system compares a bid with the current highest bid or the minPrice. If no bid is placed yet, it creates a new Bid. If the bidden price is higher than the current highest bid, the bid is marked as the new highestBid. Each bid has exactly one bidder and all bidders of an auction should be stored in the bidders collection of that auction. A String containing information of the auction’s items (see Task section for example) can be generated any time of the auction (generateItemListString()). Eventually when there are no more incoming bids the auctioneer closes the auction (closeAuction()). The closeAuction() method should return the String generated by generateItemListString(). Additional items may be registered during the auction until the auction is closed. If you try to register an Item which has the same name as one of the auction’s items, an IlleagalArgumentException is thrown. After the auction’s closed, bidding on or adding an item, as well as closing the auction again, results in an IllegalStateException.

All-pay auction

Additionally to English auctions, a second auction type is required by the auction system: In all-pay auctions every bid must be payed. As a consequence our auction system should additionally generate all bids (generateAllBidsString()) when calling generateItemString() for all-pay auctions. Implement the generateItemListString() method using the template method design pattern and name the primitive method generateItemString().

Task

Implement the auction system structure!

The generate*String() and toString() methods should be implemented in such a way that thegenerateItemListString() method returns strings like the following examples:

  1. for an EnglishAuction 

    Blueprint: A blueprint for building a touch screen (minimum bidding price: 5 EUR)
Highest bid: 10 EUR by Steve Jobs
Magazine: Computer magazine (minimum bidding price: 5 EUR)
Highest bid: 5 EUR by Linus Torvalds
Watch: jWatch (minimum bidding price: 150 EUR)
No bids placed

  2. for an AllPayAuction 

    Blueprint: A blueprint for building a touch screen (minimum bidding price:5 EUR)
Highest bid: 10 EUR by Steve Jobs
All bids:
5 EUR by Bill Gates
7 EUR by Linus Torvalds
10 EUR by Steve Jobs
Magazine: Computer magazine (minimum bidding price: 5 EUR)
Highest bid: 5 EUR by Linus Torvalds
All bids:
5 EUR by Linus Torvalds
Watch: jWatch (minimum bidding price: 150 EUR)
No bids placed

The lines must be separated by the newline character \n.

Hints

The most important erroneous parameter values must be handled properly. Therefore, the parameter value null should result in a NullPointerException while others like empty Strings, zero or negative numbers that do not make sense should cause an IllegalArgumentException.
