Inloop / Tasks / Classes and objects

In this task it is your job to implement a Taxi class that represents taxis which have a driver and seats for four passengers. Goal of this task is to give you an impression on how classes and methods in Java work and how to use them.

To solve this task follow the instructions given in the task description.

Task 1

Every passenger is represented by an instance of the class Human that has the String attributes forename and name (the surname). Every Human can be asked for his name using getName() or his forename using getForename().

Task 2

Every class in Java is derived from the basic class Object which implements a method called toString(). This method provides the String representation of a certain object of the class. It is often used implicitly in Java like in this case: 

System.out.println(new Human("Udo", "G"));

The line of code above prints a String like:

myPackage.Human@15fea60

Now override the method toString() in your Human class to get a nicer representation like: 

Human aHuman = new Human("Maximilian", "Huber");
System.out.println("This is " + aHuman + ".");

This generates the following output (the forename is separated with a space character):

This is Maximilian Huber.
Task 3

Create a class called Taxi which represents a taxi. Because every taxi has a driver, every Taxi object should have a driver attribute (of type Human). Additionally, the Taxi can have passengers who are also represented by Human objects.

When a Taxi is created by a constructor method, a driver must be assigned to it. Hence, the class Taxi needs a constructor with one parameter of type Human. You can ask for the name of the Taxi‘s driver with the method getDriverName() which gives you a String representation of the Human object created for the driver. An example for the output would be: 

Juergen Staub

Please take care: Do not return the actual Human object but the complete name of the human instead.

Each taxi can transport up to four passengers plus its driver. Humans get into the taxi separately. The Taxi class has the method add(Human x) to add a new passenger. If the taxi is full, the method doesn’t add the passenger. Instead, the following message shall be printed to the console: 

We are sorry, Lucky Fuke. The taxi is full.

Of course, the name of the respective person is used.

Your task is to implement all of this.

Task 3.1

When calling the method Taxi.toString(), the status of the Taxi is returned. Assuming the taxi is driven by Juergen Staub and he has the guests Andrea Bora, Franzi Ada, Leah Posh and Susi Fresh, one of the following messages is printed:

  1. If the taxi is empty:
    This is the taxi of Juergen Staub. He takes nobody along.
  2. If Andrea Bora is the only passenger:
    This is the taxi of Juergen Staub. He takes Andrea Bora along.
  3. If all four guests are in the taxi:
    This is the taxi of Juergen Staub. He takes Andrea Bora, Franzi Ada, Leah Posh and Susi Fresh along.

Take care of the correct construction of the passenger list.

Task 3.2

Although passengers get in separately, they can only get out all together. This happens when the method Taxi.allGetOut() (return type Human[]) is called. After that, the taxi is empty again (except for the driver). Additionally, an array (of type Human) containing the Human objects of the former passengers is returned. If there were no passengers in the taxi, an empty array is returned. The returned array must not contain null values.

Task 3.3

This task is optional but you can use the following lines of code to test your implementations of the classes Human and Taxi. Therefore you should create a new class (e.g. MyTaxi) with only one method main() and with following code:

MyTaxi.java

If you have done everything right, there should be no compile errors and you should receive the following output:
