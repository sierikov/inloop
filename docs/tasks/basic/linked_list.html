Linked List

Although there are a whole bunch of container classes (Collections) included in the Java API (in the packages java.util.*), today’s lesson is to implement your own linked list!

Your task is to write a singly-linked list without using the collections framework. Elements of this list should contain String objects.

Your implementation must contain two classes:

  1. ListElement - Represents one element of the list.
  2. List - Represents a linked list itself.
List Element

Your ListElement class must at least contain:

  1. An attribute content of type String representing the element’s content.
  2. An attribute next that references the successor element in the list.
  3. A constructor (with only one String argument) which ensures that no empty element can be created (see explanations below).

Mind visibilities (private/public) and implement a getter and setter for each attribute (getContent()/setContent(), getNext()/setNext())! Ensure that a ListElement cannot be made empty at any point!

Exceptions

Whenever you write a method with parameters, you should think about with which arguments (i.e. actual values for a method’s parameters) the method might be called.

If you fully control the arguments passed to the method and you only pass arguments that are valid in terms of what the method expects, you should be fine.

But in general, it is always advisable and considered good practice to program defensively. This not only holds true for code that will be used by other people (known as public API). Checking for valid arguments allows a program to fail as early as possible (fail-fast approach), close to the actual cause of an error. This makes debugging programs a lot easier. Do not be tempted to skip such checks for the sake of perceived better performance (quoting Donald Knuth: Premature optimization is the root of all evil.).

If your code can possibly use a fallback mechanism in case of invalid arguments, it should do so. Otherwise, the program’s execution cannot be continued properly; an exception occurs.

In Java, there are classes that represent exceptional events of a program’s execution: Throwable and its subclasses Error and Exception. For now, we will focus on two subclasses of Exception. Those two are so called unchecked exceptions.

  1. The IllegalArgumentException is usually thrown at the beginning of a method if any of the arguments is invalid in terms of what the method expects and needs to work correctly.
  2. The NullPointerException indicates that a variable contains null when instead it should contain a reference to an actual object. The runtime environment throws a NullPointerException when code tries to access the object in a variable that actually contains null (a so called null dereference). APIs usually throw this type of Exception when an argument is null.

The latter case is a special case of an invalid argument so throwing a NullPointerException instead of an IllegalArgumentException shows directly why the argument is invalid.

Exceptions can be manually thrown by using the throw keyword followed by an instance of Exception. Optionally, on creation of a new Exception object, you can also specify a message giving detailed information about the exception. Assuming a String variable content that cannot contain null or the empty String (""), you could do the following: 

if (content.isEmpty()) {
    throw new IllegalArgumentException("The argument 'content' cannot be empty!");
}

Objects.requireNonNull(Object) takes an arbitrary object as argument and either throws a NullPointerException (if the argument actually was null) or returns the object (otherwise).

List

The List class should be able to:

  1. append a new element to the end of the list with the method append(String string) where string is the content of the new element (remember to check the parameter!).
  2. remove an element with the method remove(String string) (remove the first element whose content equals string).
  3. remove() must return the removed element’s content or null if no element was removed!

Be careful with the special cases of an empty list and a list with only one element. Think about how to link the list to its elements. You may find help in the material of lectures held in your previous semesters.

Test your implementation for yourself with your own main class but only submit the two classes List and ListElement!

Hints

The most important erroneous parameter values must be handled properly. Therefore, the parameter value null should result in a NullPointerException while others like empty Strings at points where they cannot be handled properly should cause an IllegalArgumentException.
